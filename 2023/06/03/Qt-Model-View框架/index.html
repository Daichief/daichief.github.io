MVD框架

1.MVC框架

  • 模型-视图-控制(MVC) 是源自 Smalltalk 的一种设计模式,在构建用户界面时经常使用
  • MVC 框架的目的是用于将业务逻辑、数据、界面显示分离的方法，在修改界面时不会影响业务逻辑
  • MVC 框架由三个对象组成:
    • Model (模型) : 用于封装与应用程序业务逻辑相关的数据以及对数据的处理方法。Model 有对数据直接访问的权力
    • View (视图) ： 用于数据可视化
    • Controller (控制器 ） : 控制器作用于模型和视图上。它控制数据流向模型对象，并在数据变化时更新视图。它使视图与模型分离开。

2.Qt Model/View框架

  • Qt 中的 Model/View 就是基于 MVC 设计思想设计,只不过 Controller 变成了 Delegate
  • MVD 框架主要包含两个部分
    • Model : 模型,负责存储和管理数据
    • View : 视图,负责显示数据
    • Delegate : 委托,负责渲染和编辑

3.视图组件（View）

  • 用于显示数据模型的数据界面的组件,Qt 提供的组件如下:

    • QListView：用于显示单列的列表数据，适用于一维数据的操作。

    • QTreeView：用于显示树状结构数据，适用于树状结构数据的操作。

    • QTableView：用于显示表格状数据，适用于二维表格型数据的操作。

    • QColumnView：用于多个QListView显示树状层次结构，树状结构的一层用一个QListView显示。

    • QHeaderView：提供行表头或列表头的视图组件，如QTableView的行表头和列表头。

4.模型（Model）

  • 模型负责对数据进行管理,实现通用接口,为 delegate，view类对象访问和修改数据提供支持。
  • 数据模型的基本形式是使用行和列定义的表格数据,每个数据项(Item）按照其行号和列号定位

5.模型索引（QModelIndex）

  • 为了保证数据的表示与数据存取方式隔离，数据模型中引入了模型索引的概念。

  • 通过数据模型存取的每个数据都有一个模型索引，视图组件和代理都通过模型索引来获取数据

  • QModeIndex 类表示模型索引的类,模型索引提供数据存取的一个临时指针,用于通过数据模型提取或修改数据

6.标准数据项模型

  • QStandardItemModel 是标准的以项 (item) 数据为单位的基于M/V模型的一种标准数据管理方式
  • 一个数据模型中一个数据就是一个数据项(Item)
  • 在 Qt 中使用 QStandardItem 来描述标准数据项

  • 在模型-视图中每个数据项包含一个或多个由角色（Role）标识的数据值。
  • 角色是一个整数值，用于标识数据项中的不同数据。

Qt::DisplayRole 显示数据的角色，显示在视图中的单元格中
Qt::EditRole 设置数据项的编辑数据，可以方便地对数据进行编辑和更新

8.Qt 自定义Delegate

  • 支持 mvd 模式的组件，如 QTableView,QTreeView 都是支持默认委托，用于基本的渲染,如显示文本、数字和图像基本数据类型

  • 默认委托一般使用的 QStyledItemDelegate ,当默认委托无法满足业务需求时,则需要进行自定义

  • QStyledItemDelegate 的父类 是 QAbstractItemDelegate
